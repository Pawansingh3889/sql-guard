## Title

`Who's using sql-sop? Tell us about your setup`

## Body

sql-sop's install count says people are using it, but the repo never hears
about most of those use cases. If sql-sop lives in your CI or pre-commit
hook, a one-liner reply below genuinely helps:

- **Where is it running?** pre-commit, GitHub Action, Makefile, manual?
- **How big is the codebase?** rough SQL file count, or "dozens", "hundreds".
- **Which rule catches the most?** E001 `delete-without-where`, W001
  `select-star`, P004 `bare-variable-in-execute`, something else?
- **Which rule would you love to have next?** link a
  [rule-request issue](https://github.com/Pawansingh3889/sql-guard/issues/new?template=rule-request.yml)
  or just describe.

Anonymous or behind a pseudonym is fine. Named replies get featured in
the README's "Used by" section (with permission).

Low-stakes, low-effort - but every reply is a data point that shapes
what lands in v0.5.

Thanks for being here.
